# Project Summary
The Smart Supermarket Management System is an AI-powered platform designed to optimize supermarket operations and improve business efficiency. It provides retailers with advanced tools for operational management, data analytics, and informed decision-making. The system includes modules such as Point of Sale (POS), accounting features, and management tools for customers and employees, making it ideal for large-scale commercial deployments.

# Project Module Description
The project consists of several functional modules:
- **Backend**: Handles data storage, user authentication, and API endpoints for frontend interaction.
- **Frontend**: Offers a user-friendly interface for supermarket operations, including dashboards for sales, inventory, customer, and employee management.
- **Point of Sale (POS)**: Manages real-time sales transactions and inventory updates with enhanced error handling.
- **Supplier Management**: Facilitates CRUD operations for supplier data.
- **Purchase Order Management**: Manages the lifecycle of purchase orders, including tracking and status updates.
- **Customer Management**: A CRM system for maintaining customer profiles, loyalty programs, and analytics.
- **Employee Management**: HR module for managing employee records and payroll.
- **Advanced Reports**: Provides analytics and reporting capabilities across various business dimensions.
- **Settings**: Manages system settings, including a feature for resetting demo data.

# File Description Inventory
- **backend/**: Contains all server-side code, including API routes, data models, and services.
- **frontend/**: Contains the client-side application, including components, pages, and assets.
- **requirements.txt**: Lists Python dependencies for backend development.
- **package.json**: Lists JavaScript dependencies for frontend development.
- **src/types/index.ts**: Type definitions for users, products, sales, and accounting.
- **src/lib/api.ts**: API helper functions for interacting with backend services.
- **src/pages/Login.tsx**: Secure login interface with validation.
- **src/pages/Customers.tsx**: Customer management interface with CRUD functionalities and improved error handling.
- **src/pages/Employees.tsx**: Employee management interface with CRUD functionalities.
- **src/pages/AdvancedReports.tsx**: Comprehensive analytics system for generating various reports.
- **src/pages/Settings.tsx**: Settings page for managing demo data and viewing database status.
- **src/pages/POS.tsx**: Updated POS interface with improved error handling and loading states.
- **src/pages/PurchaseOrders.tsx**: Purchase order management interface with CRUD functionalities and improved loading states.
- **src/pages/Suppliers.tsx**: Supplier management interface with CRUD functionalities and improved loading states.
- **core/config.py**: Configuration file that includes default values for database URL and other settings.

# Technology Stack
- **Backend**: Python, FastAPI, SQLAlchemy
- **Frontend**: React, TypeScript, Vite, Tailwind CSS
- **Database**: PostgreSQL for data persistence and management

# Usage
1. **Install Dependencies**:
   - For backend: `pip install -r requirements.txt`
   - For frontend: `pnpm install`
   
2. **Run the Application**:
   - Start the backend server.
   - Build and serve the frontend application.

3. **Seeding the Database**:
   - Run the command `python3 scripts/seed_database.py` in the backend directory to populate the database with sample data.
